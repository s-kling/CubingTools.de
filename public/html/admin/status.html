HTTP Status Codes

2xx — Success
  • 200 OK: Request successful. The server processed it correctly and returned the requested data.
  • 201 Created: Resource created successfully (e.g., new user added). Returned on successful POST requests that create resources.
  • 204 No Content: Request successful but no content to return. Typical for successful DELETE, OPTIONS, or acknowledged requests.
3xx — Redirection
  • 302 Found: Temporary redirect to another URL. Used after email confirmation or when a resource has moved temporarily.
4xx — Client Error
  • 400 Bad Request: Invalid request format, missing required fields, or validation failed. Check JSON syntax, field lengths, and email format.
  • 401 Unauthorized: Authentication failed, token missing, expired, or invalid. Re-login or refresh your session token.
  • 403 Forbidden: Authenticated but insufficient permissions for this resource. Your role (tester/operator/admin) lacks access.
  • 404 Not Found: Endpoint or resource doesn't exist. Verify the API path and resource ID.
  • 409 Conflict: Action conflicts with existing state. For example, a username already exists or an appeal is already pending.
  • 418 I'm a teapot: The server refuses to brew coffee because it is, permanently, a teapot.
  • 429 Too Many Requests: Rate limit exceeded. Too many requests in a short time window. Wait before retrying.
5xx — Server Error
  • 500 Internal Server Error: Backend error during processing. Check server logs and contact administrators if it persists.
